## Build Commands

This is an ESP-IDF 6.0.1 project targeting the Seeed Studio XIAO ESP32S3 board.

```bash
# Activate ESP-IDF environment
export IDF_PATH=/home/relog/.espressif/v6.0.1/esp-idf
export IDF_TOOLS_PATH=/home/relog/.espressif/tools
export ESP_IDF_VERSION=6.0.1
export IDF_PYTHON_ENV_PATH=$IDF_TOOLS_PATH/python/v6.0.1/venv
export PATH=$IDF_TOOLS_PATH/ninja/1.12.1:$IDF_TOOLS_PATH/xtensa-esp-elf/esp-15.2.0_20251204/xtensa-esp-elf/bin:$IDF_PATH/tools:$IDF_PYTHON_ENV_PATH/bin:$PATH

# Full build
idf.py build

# Clean build
idf.py fullclean && idf.py build

# Flash to device (serial port assumed /dev/ttyACM0)
idf.py flash

# Flash + open serial monitor
idf.py flash monitor

# Serial monitor only
idf.py monitor

# Menuconfig
idf.py menuconfig
```

Compile commands are in `build/compile_commands.json`, used by clangd for LSP support. `.clangd` strips `-f*` and `-m*` flags that the ESP32 clangd toolchain can't interpret.

## Architecture

This is an IoT device that boots, connects to WiFi, syncs time via NTP, fetches weather from wttr.in, and sends a boot-up notification to a DingTalk (钉钉) chat group via webhook. It also monitors indoor environment via a BME680 sensor and sends threshold-based alerts through DingTalk.

**`main/main.c`** — App entry point (`app_main`). Runs a linear startup sequence:

1. **WiFi STA init** (`wifi_init_sta`): Initializes NVS, creates a FreeRTOS event group, sets up the netif stack, registers WiFi/IP event handlers, and connects as a station. Uses `WIFI_CONNECTED_BIT` to signal `app_main` once a DHCP IP is obtained.
2. **SNTP time sync** (`obtain_time`): Polls pool.ntp.org and ntp.aliyun.com, blocks up to 30 s for sync.
3. **Weather fetch** (`fetch_weather`): HTTP GET to `wttr.in/{city}?format=j1`, accumulates response in a static buffer, parses fields via a minimal hand-rolled JSON extractor (`json_get_str`).
4. **DingTalk notification** (`dingtalk_send`): Sends a boot-up text message via DingTalk webhook.
5. **BME680 init** (`bme680_init`): Initializes I2C bus (GPIO4=SDA, GPIO5=SCL), verifies chip, reads calibration data, configures oversampling and gas heater.
6. **Environment monitor** (`env_monitor_start`): Creates a FreeRTOS task that reads the sensor every 60 s and sends DingTalk alerts on threshold crosses.

**`main/bme680.c`** — BME680 I2C sensor driver. Uses the legacy `driver/i2c.h` API. Reads two calibration blocks (0x89–0xA1, 0xE1–0xF0) plus gas range registers (0x00, 0x02, 0x04) and gas LUT (0x08–0x1F). Implements temperature, pressure, humidity, and gas resistance compensation per the Bosch datasheet. Configures gas heater profile for VOC sensing (~320 °C plate, 150 ms).

**`main/bme680.h`** — Driver header. Defines I2C pins (`GPIO_NUM_4`/`GPIO_NUM_5`), address (`0x76`), and the `bme680_data_t` struct (temperature, humidity, pressure, gas_resistance).

**`main/env_monitor.c`** — Indoor environment scene monitor. FreeRTOS task at 60 s interval. Tracks pressure trend over a 60-sample circular buffer (1 h) for rain prediction. Six alert types with 30-min cooldown each: temp high/low, humidity high/low, pressure drop, VOC high. Combo alert when ≥2 thresholds fire simultaneously. Calls `dingtalk_send()` for notifications.

**`main/env_monitor.h`** — Monitor header. Defines `env_threshold_t` struct and default thresholds (`ENV_DEFAULT_TEMP_LOW=10°C`, `ENV_DEFAULT_TEMP_HIGH=33°C`, `ENV_DEFAULT_HUMIDITY_LOW=20%`, `ENV_DEFAULT_HUMIDITY_HIGH=75%`, `ENV_DEFAULT_PRESSURE_DROP=2hPa`, `ENV_DEFAULT_GAS_RES_LOW=50kΩ`).

**`main/dingtalk_notify.c`** — DingTalk robot webhook client. Constructs a POST to `https://oapi.dingtalk.com/robot/send` with token from `DINGTALK_ACCESS_TOKEN`. Message format: `{"msgtype":"text","text":{"content":"..."}}`. If token is still the placeholder value, the call is a no-op with a warning.

**`main/dingtalk_notify.h`** — Public API header. Contains the `DINGTALK_ACCESS_TOKEN` and `DINGTALK_SECRET` defines (the latter for HMAC signing, currently unused).

## Configuration

- **WiFi credentials**: `WIFI_SSID` / `WIFI_PASSWORD` in `main/wifi_app.c`
- **DingTalk token**: `DINGTALK_ACCESS_TOKEN` in `main/dingtalk_notify.h`
- **Weather city**: `WEATHER_CITY` in `main/weather.c`
- **I2C pins / BME680 addr**: `BME680_I2C_SDA_IO` / `BME680_I2C_SCL_IO` / `BME680_I2C_ADDR` in `main/bme680.h`
- **Alert thresholds**: `ENV_DEFAULT_*` macros in `main/env_monitor.h`
- **ESP-IDF SDK config**: `sdkconfig` (generated by `idf.py menuconfig` — don't hand-edit)

## Known Issues

- The legacy I2C driver (`driver/i2c.h`) is deprecated as of ESP-IDF v6.0 and will be removed in v7.0. Migrate to `driver/i2c_master.h` before upgrading.
- `get_idf` shell alias is assumed but not guaranteed — fall back to the manual `export` commands listed under Build Commands if the alias isn't available.
