## Build Commands

This is an ESP-IDF 6.0.1 project targeting the Seeed Studio XIAO ESP32S3 board. The IDF_PATH environment must be set before building.

```bash
# Activate ESP-IDF environment (typically alias'd in your shell)
get_idf

# Full build
idf.py build

# Clean build
idf.py fullclean && idf.py build

# Flash to device (serial port assumed /dev/ttyACM0)
idf.py flash

# Flash + open serial monitor
idf.py flash monitor

# Serial monitor only
idf.py monitor

# Menuconfig
idf.py menuconfig
```

Compile commands are in `build/compile_commands.json`, used by clangd for LSP support. `.clangd` strips `-f*` and `-m*` flags that the ESP32 clangd toolchain can't interpret.

## Architecture

This is a simple IoT device that boots, connects to WiFi, syncs time via NTP, fetches weather from wttr.in, and sends a boot-up notification to a DingTalk (钉钉) chat group via webhook.

**`main/main.c`** — App entry point (`app_main`). Runs a linear startup sequence:

1. **WiFi STA init** (`wifi_init_sta`): Initializes NVS, creates a FreeRTOS event group, sets up the netif stack, registers WiFi/IP event handlers, and connects as a station. Uses `WIFI_CONNECTED_BIT` to signal `app_main` once a DHCP IP is obtained.
2. **SNTP time sync** (`obtain_time`): Polls pool.ntp.org and ntp.aliyun.com, blocks up to 30 s for sync.
3. **Weather fetch** (`fetch_weather`): HTTP GET to `wttr.in/{city}?format=j1`, accumulates response in a static buffer, parses fields via a minimal hand-rolled JSON extractor (`json_get_str`).
4. **DingTalk notification** (`dingtalk_send`): Sends a text message via DingTalk webhook (HTTP POST with JSON body).

**`main/dingtalk_notify.c`** — DingTalk robot webhook client. Constructs a POST to `https://oapi.dingtalk.com/robot/send` with token from `DINGTALK_ACCESS_TOKEN`. Message format: `{"msgtype":"text","text":{"content":"..."}}`. If token is still the placeholder value, the call is a no-op with a warning.

**`main/dingtalk_notify.h`** — Public API header. Contains the `DINGTALK_ACCESS_TOKEN` and `DINGTALK_SECRET` defines (the latter for HMAC signing, currently unused).

## Configuration

- **WiFi credentials**: `WIFI_SSID` / `WIFI_PASSWORD` in `main/main.c`
- **DingTalk token**: `DINGTALK_ACCESS_TOKEN` in `main/dingtalk_notify.h`
- **Weather city**: `WEATHER_CITY` in `main/main.c`
- **ESP-IDF SDK config**: `sdkconfig` (generated by `idf.py menuconfig` — don't hand-edit)
